# Phase 7 Submission And Pitch Prep (Hours 34-36)

## 1. Devpost Submission Requirements

- 3 to 5 minute demo video
- project description
- technologies used
- GitHub repo link
- live demo link
- video link

## 2. Video Recording Checklist

- Use OBS, Screen Studio, or Loom
- Record at `1920x1080`
- Use 30 FPS or 60 FPS
- Check microphone level before recording
- Record in a quiet room
- Do one full rehearsal before the final take

## 3. Shark Tank Pitch Outline

### Slide 1: The problem

- open applicant pools create abuse and manual review
- spreadsheets do not solve identity or automation

### Slide 2: The solution

- live demo of organizer flow and claimant flow

### Slide 3: Why World ID matters

- nullifiers make one-person-one-claim enforceable
- this is structural, not decorative login

### Slide 4: Scale and impact

- hackathons today
- rebates, grants, nonprofits, utility programs tomorrow

### Slide 5: Team

- who built product, frontend, backend, demo

### Likely judge questions

- Why World ID instead of email?
- How do you handle duplicate receipts?
- What happens if extraction is wrong?
- Can this scale beyond hackathons?

## 4. GitHub Repo Polish

- README with screenshots and setup
- `LICENSE`
- `.env.local.example`
- architecture overview
- concise commit history if time allows

## 5. Final Submission Checklist

- deployed app works
- demo data loaded
- README polished
- video uploaded
- Devpost form complete
- pitch talking points practiced
